# Core Persona Deck

## Purpose

Use this universal review bench for every plan-required task before adding
class-specific personas.

## Universal Review Bench

| Persona | Plan-improvement mandate |
| --- | --- |
| Intent and Output Contract Reviewer | Ensure the plan solves the user's actual request and names the final answer or artifact shape. |
| Scope and Surface Reviewer | Make affected files, systems, accounts, docs, APIs, datasets, and stakeholders explicit. |
| Assumption Breaker | Identify guessed facts, permissions, inputs, versions, paths, user preferences, and prerequisites. |
| Risk and Reversibility Reviewer | Separate reversible steps from hard-to-reverse steps and add rollback or recovery paths. |
| Verification Oracle Reviewer | Add tests, citations, calculations, previews, smoke checks, or acceptance criteria that match the risk. |
| Source and Freshness Reviewer | Require the right source authority and current information where needed. |
| Simplicity and Cost Reviewer | Remove unnecessary steps, reviewers, tools, and verification that exceed the task risk. |
| User Gate and Consent Reviewer | Add explicit user approval before destructive, paid, public, account-changing, or external actions. |
| Handoff and Continuity Reviewer | Ensure future users or agents can resume from durable decisions, evidence, and next actions. |

## Minimum Use

- Medium-risk plan: use the universal bench plus 2 to 4 class-specific personas.
- High-risk plan: use the universal bench plus the full class-specific deck.
- Multi-class high-risk plan: use the universal bench plus only non-overlapping
  class-specific personas from each class.

Never add a persona that cannot produce a concrete plan change.
